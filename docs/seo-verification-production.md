# SEO verification checklist (production)

Use this after deploying to **https://extensionshield.com** to confirm indexing, meta tags, and Search Console setup.

---

## 1. Sitemap & robots (curl)

### Sitemap — 200 and valid XML

```bash
curl -i https://extensionshield.com/sitemap.xml
```

**Check:**

- **Status:** `200 OK`
- **Body:** Valid XML with `<urlset>`, multiple `<url>` entries. **Every** `<url>` must include:
  - `<loc>`
  - **`<lastmod>`** (YYYY-MM-DD; generator uses sitemap build date so it’s consistent)
  - `<changefreq>`
  - `<priority>`
- **URLs:** Include key routes (e.g. `/`, `/about`, `/research`, `/research/benchmarks`, `/privacy-policy`, `/enterprise`, `/scan`, etc.)

If `<lastmod>` is missing on production, redeploy so the latest `public/sitemap.xml` (generated by `npm run generate:sitemap`) is served.

### robots.txt — includes Sitemap line

```bash
curl https://extensionshield.com/robots.txt
```

**Check:**

- Line: `Sitemap: https://extensionshield.com/sitemap.xml`
- `User-agent: *` and `Allow: /`
- `Disallow: /settings` and `Disallow: /reports` (optional but recommended)

**Note:** In production, Cloudflare may append **Managed Content** blocks (including disallow rules for some AI crawlers, e.g. Google-Extended). That is expected and does not affect normal Google Search (Googlebot) crawling. The repo’s `frontend/public/robots.txt` is the base; Cloudflare adds its blocks at serve time.

### Sitemap/robots fetch reliability (WAF / bot protection)

If a **web tool or script** (or Googlebot) gets **4xx** (e.g. 400, 403, 429) when fetching `https://extensionshield.com/sitemap.xml` or `https://extensionshield.com/robots.txt`, while the **browser** loads them fine, the cause is usually **Cloudflare WAF or bot protection** challenging non-browser requests.

- **Symptom in Google Search Console:** “Sitemap could not be read” or “Couldn’t fetch” for the sitemap URL.
- **Fix (recommended):** In **Cloudflare** → **Security** → **WAF** → **Custom rules**, add a rule that **allows** requests to `/sitemap.xml` and `/robots.txt` when the client is a **verified bot** (e.g. `cf.client.bot`), so Googlebot and other good crawlers are not challenged or blocked.

Example rule idea (adapt to your Cloudflare UI):

- **Name:** e.g. “Allow verified bots for sitemap and robots”
- **Expression:** `(http.request.uri.path eq "/sitemap.xml" or http.request.uri.path eq "/robots.txt") and cf.client.bot`
- **Action:** Skip (or Allow) — skip WAF/challenge for this combination.

Result: Googlebot and other verified bots can fetch sitemap and robots without challenge; other traffic is unchanged.

## 2. Page source (meta tags)

For each of **`/about`**, **`/research`**, and **`/privacy-policy`**:

1. Open the URL in the browser.
2. **View Page Source** (right‑click → View Page Source, or Ctrl/Cmd+U).
3. In the `<head>` section, confirm:

| Check | What to look for |
|-------|-------------------|
| **Unique `<title>`** | One `<title>` tag; text matches the page (e.g. "About Us \| ExtensionShield", "Extension Threat Research & Case Studies \| ExtensionShield", "Privacy Policy \| ExtensionShield"). |
| **Meta description** | One `<meta name="description" content="...">` with the correct description for that page. |
| **Canonical** | One `<link rel="canonical" href="https://extensionshield.com/...">` with the exact URL for that path (no trailing slash except for `/`). |
| **OG tags** | `<meta property="og:title">`, `og:description`, `og:image`, `og:url`, `og:type`, and ideally `og:image:width`, `og:image:height`. |
| **Twitter tags** | `<meta name="twitter:card">`, `twitter:title`, `twitter:description`, `twitter:image` (and optionally `twitter:site`). |

**Note:** In an SPA, View Source shows the initial HTML. If meta is injected by React/Helmet, use **Inspect** (DevTools → Elements) after the page has loaded and check `<head>` there for the same tags.

## 3. Google Search Console

### Submit sitemap

1. Go to [Google Search Console](https://search.google.com/search-console).
2. Select the property for **https://extensionshield.com** (or the correct property).
3. **Sitemaps** (left menu) → enter: `https://extensionshield.com/sitemap.xml` → **Submit**.
4. Confirm status becomes “Success” (or “Sitemap could not be read” is resolved).

### URL inspection (indexing)

For each of:

- **https://extensionshield.com/about**
- **https://extensionshield.com/research/benchmarks**
- **https://extensionshield.com/privacy-policy**

1. **URL Inspection** (left menu or top search bar).
2. Paste the full URL → **Enter** (or “Request indexing” if you just submitted).
3. **Check:**
   - **URL is on Google:** “URL is on Google” / **Indexed**.
   - **Crawling:** “Last crawl” shows a recent date; “Crawled as” shows **Googlebot** (or Googlebot Smartphone).
   - **Rendered HTML:** Use “View tested page” / “Screenshot” or “More info” → “View crawled page” (if available). Confirm the **rendered HTML** contains the main content (headings, body text) for that page, not only the shell from `index.html`.

If a URL is not yet indexed, use **Request indexing** and re-check after Google recrawls (can take hours to days).

## 4. Quick reference

| Item | Command or action | Pass criteria |
|------|-------------------|----------------|
| Sitemap HTTP | `curl -i https://extensionshield.com/sitemap.xml` | 200, valid XML |
| robots.txt | `curl https://extensionshield.com/robots.txt` | Contains `Sitemap: https://extensionshield.com/sitemap.xml` |
| Sitemap/robots for bots | Cloudflare WAF | Rule to allow verified bots (`cf.client.bot`) for `/sitemap.xml` and `/robots.txt` so GSC can fetch |
| /about, /research, /privacy-policy | View source or Inspect `<head>` | Unique title, meta description, canonical, OG + Twitter tags |
| Search Console | Sitemaps → Submit | sitemap.xml submitted and success |
| Search Console | URL Inspection for /about, /research/benchmarks, /privacy-policy | Indexed, crawled as Googlebot, rendered HTML has main content |

**If GSC shows “Sitemap could not be read” or “Couldn’t fetch”:** see [Sitemap/robots fetch reliability](#sitemaprobots-fetch-reliability-waf--bot-protection) — add a Cloudflare WAF rule so verified bots (e.g. Googlebot) are not challenged on `/sitemap.xml` and `/robots.txt`.
