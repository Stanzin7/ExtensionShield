## 4. Sitemap vs Route Config

**Sitemap** is built from a hardcoded list in `frontend/scripts/generate-sitemap.js`, not from `getSitemapRoutes()` in `routes.jsx`. The following routes have `seo` in `routes.jsx` but are **missing** from the current `sitemap.xml`:

- `/about`
- `/community`
- `/privacy-policy`
- `/research/benchmarks`

**Evidence:**

```368:374:frontend/src/routes/routes.jsx
export const getSitemapRoutes = () => {
  return routes.filter(route =>
    route.seo &&
    !route.path.includes(":") &&
    !route.path.includes("*")
  );
};
```

`frontend/public/sitemap.xml` contains 17 URLs; `getSitemapRoutes()` would return more (e.g. about, community, privacy-policy, research/benchmarks).

## 5. OG Image

`seoUtils.getOGImage()` returns `https://extensionshield.com/logo.png`. In the repo, `frontend/public/` does **not** contain `logo.png` (only favicons, manifest, sitemap, robots, _headers, etc.). If not added at deploy time, OG image requests will 404.

```29:33:frontend/src/utils/seoUtils.js

  // For now, use the same logo for all pages
  return `${CANONICAL_DOMAIN}/logo.png`;
```

## 6. Render / Indexability (SPA)

- **Rendering:** Content is client-rendered only. No SSR or prerender in Vite config.
- **Critical pages (/, /scan, /research/*, /enterprise):** Require JS to render main content. Crawlers that execute JS will see updated `<title>` and meta when Helmet/SEOHead run; crawlers that do not execute JS will see only `index.html` (single title/description, no per-URL canonical/OG).
- **Pre-rendering / static HTML:** None in the codebase.

## 8. What Is Currently Missing (Consolidated)

- **Canonical:** Missing on `/privacy-policy` and on dynamic/utility routes that may be linked (e.g. scan results). Index.html has no canonical (acceptable for SPA if default is homepage).
- **Per-route title/description:** Missing for Privacy Policy and for scan progress/results, settings, reports, auth.
- **OpenGraph (og:title, og:description, og:image, og:url, og:type/site_name):** Missing on all pages that use only `Helmet` (research, about, community, gsoc, open-source); missing on privacy-policy and dynamic routes.
- **Twitter card tags:** Same as OG — only present where `SEOHead` is used (home, scan, scan/history, enterprise, glossary).
- **Meta robots:** Not set on indexable pages (good). Could be set to `noindex` on `/settings`, `/reports`, auth callbacks if those should not be indexed.
- **Structured data (JSON-LD):** Present only where explicitly passed (e.g. HomePage, ScannerPage, EnterprisePage, GlossaryPage). Other key pages (research, about, privacy-policy) have none.
- **Sitemap coverage:** Missing `/about`, `/community`, `/privacy-policy`, `/research/benchmarks`.
- **Reliable og:image:** `/logo.png` not in repo; need to add asset or change URL to an existing image (e.g. favicon-512).
