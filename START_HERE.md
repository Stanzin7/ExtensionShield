## ✅ 5-Minute Quick Start

**Without reading anything, here's what to do:**

```bash
# 1. Go to https://app.supabase.com

# 2. Create project
#    Name: ExtensionShield
#    Password: anything strong
#    Region: closest to you
#    Wait 2-3 minutes

# 3. Copy credentials from Settings > API
#    Save:
#    - Project URL
#    - anon public key

# 4. Create frontend/.env file
cd /Users/stanzin/Desktop/ExtensionShield/frontend
touch .env

# 5. Paste these into .env:
# VITE_SUPABASE_URL=https://your-url.supabase.co
# VITE_SUPABASE_ANON_KEY=your-anon-key

# 6. Run the app
npm run dev

# 7. Open http://localhost:5173
# Click Sign In
# Test with: test@example.com / TestPassword123
```

**Done! ✅**

## 💡 Key Points to Remember

1. **Project URL** - Where Supabase lives
2. **Anon Key** - Public API key (NOT service_role)
3. **.env file** - Goes in `frontend/` folder (NOT root)
4. **Variable names** - Must start with `VITE_` (case sensitive)
5. **Restart server** - Always restart after creating/updating .env
6. **Test immediately** - Sign In button should work right away
